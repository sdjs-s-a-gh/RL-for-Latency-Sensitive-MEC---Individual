<div align="center">
  <h1>Reinforcement Learning for Latency-Sensitive Resource Allocation in Mobile Edge Computing</h1>
</div>

This repository contains an OMNeT++ simulation that was developed as part of my BSc Computer Science final-year dissertation. The dissertation, and by extension this project, investigated how reinforcement learning could improve latency-sensitive resource allocation in Mobile Edge Computing (MEC) compared to traditional, static approaches. In other words, the project focused on seeing how an algorithm that dynamically allocates resources under different scenarios (learned through past experience) could reduce latency in comparison to a one that adopted a fixed ruleset and would not adapt.

To this end, the simulation modelled a heterogenous, 25-device environment in which devices send tasks (consisting of CPU Cycles required to be processed) to an edge server. There were a total of four different devices types that each had different boundaries for randomly generating task sizes - including Mobile Phones, Autonomous Vehicles, Augmented/Virtual Reality users (AR/VR) and Internet of Things devices (IoTs). When tasks would reach the edge server, a Proximal Policy Optimisation (PPO) agent that was trained to minimise long-term latency under varying network traffic levels subsequently allocated computational resource (a slice of CPU frequency/capacity from a shared pool). Performance metrics that included latency, energy consumption and resource utilisation were collected and analysed to compare the effectiveness of both approaches.

---
<h2>Network Topology</h2>
The simulation employed a three-tier architecture topology that covered a 1,000m x 1,000m area, which consisted of the following:
<ol>
  <li>User Equipment (UE): Heterogeneous mobile devices that moved around the simulation area, generating tasks for the edge server to process.</li>
  <li>Edge: A single basestation that wirelessly received the device-generated tasks, forwarding them to the accompanying edge server.</li>
  <li>Cloud: A remote cloud server that is only employed should the edge server's queue of tasks exceed 50.</li>
</ol>

<h2>Simulation Workflow and Technical Interaction with PPO</h2>
Rather than using a native C++ machine learning library, the PPO agent implemented in this simulation was developed using the Python library PyTorch. This decision was made as, at the time, I was a complete beginner with ML and PPO, so I wanted to give myself an easier platform to learn with. As a result, I wished to used a programming language that was more familiar to me and more readable, as well as the reduced overhead from needing to manually manage memory. However, this facet meant the interaction with OMNeT++ was not so straightforward, instead requiring code to be translated between C++ and Python using Pybind.
