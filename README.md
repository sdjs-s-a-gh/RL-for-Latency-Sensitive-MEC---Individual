This repository contains an OMNeT++ simulation that was developed as part of my BSc Computer Science final-year dissertation. The dissertation, and by extension this project, investigated how reinforcement learning could improve latency-sensitive resource allocation in Mobile Edge Computing (MEC) compared to traditional, static approaches. In other words, the project focused on seeing how an algorithm that dynamically allocates resources under different scenarios (learned through past experience) could reduce latency in comparison to a one that adopted a fixed ruleset and would not adapt.

To this end, the simulation modelled a heterogenous, 25-device environment in which devices send tasks (consisting of CPU Cycles required to be processed) to an edge server. There were a total of four different devices types that each had different boundaries for randomly generating task sizes - including Mobile Phones, Autonomous Vehicles, Augmented/Virtual Reality users (AR/VR) and Internet of Things devices (IoTs). When tasks would reach the edge server, a Proximal Policy Optimisation (PPO) agent subsequently allocated computational resource (a slice of CPU frequency from a shared pool) with the aim of minimising long-term latency. Tasks were then placed in a First-In First-Out (FIFO) queue and waited for processing.
